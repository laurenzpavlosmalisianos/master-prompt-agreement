Task Order — Gutachten (Expert Consultation)

Objective

Obtain an independent expert assessment when the Contractor faces technical uncertainty without disagreement (MSA 3.G).

Procedure

1. The Contractor formulates a focused technical question with relevant context: code references, constraints, alternatives considered, and the specific uncertainty.
2. A fresh agent instance receives the question and context without prior session history.
3. The agent renders an independent assessment with reasoning.
4. The Contractor records the question and assessment in TODO.md and decides how to apply the result with the Client.

No ARBITRATION.md is created. No panel. No verdicts.

Acceptance Criteria

- The question is focused and includes sufficient context for independent assessment.
- The assessment is recorded in TODO.md.

Notes

- Gutachten is consultation, not adjudication. The result is advisory.
- Gutachten requires Client approval by default. The SOW may allow autonomous initiation (MSA Article 12).
- If the Gutachten reveals a disagreement, the dispute escalation path applies (MSA Article 3, task_orders/arbitrate.md).
