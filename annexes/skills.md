Annex B — Contractor Qualifications (SKILLS.md)

_In consultancy framework agreements, this annex would list the consultant's certifications, specializations, and billable rate. Here it defines the Contractor's technical capabilities and operational boundaries._

Primary Specializations

[List what this Contractor is best at. Examples:]

- Python (systems programming, CLI tools, data pipelines)
- Rust (memory-safe systems, CLI applications)
- TypeScript (full-stack, strict mode)
- Static site generators (Hugo, Astro)
- SQL (PostgreSQL, query optimization)

Secondary Capabilities

[List supporting skills. Examples:]

- Shell scripting (bash, POSIX)
- Git (history management, conflict resolution, bisect)
- Docker (containerization, compose)
- CI/CD (GitHub Actions, GitLab CI)
- Technical writing (documentation, READMEs, changelogs)

Operational Capabilities

[Define what the Contractor can do autonomously. Examples:]

- Read and write files in the project directory
- Execute shell commands (build, test, lint, deploy with approval)
- Search the web for documentation and references
- Launch subagents for parallel research, arbitration (Article 3, Tier 2), or Gutachten (Article 3.G)

Known Limitations

[Be honest about what the Contractor cannot do. Examples:]

- Cannot verify visual output (screenshots, browser rendering). Flag as manual acceptance items.
- Cannot access authenticated services without credentials provided by the Client.
- Cannot persist memory across sessions except through TODO.md and project files.
- Context window is finite. For large codebases, may need to work in focused slices.

Notes

- This annex is optional. If omitted, the Contractor operates with its default capabilities.
- Update this annex when the Contractor's model or tooling changes.
- The SOW's Definitions table identifies the specific model. This annex describes what that model can do.
